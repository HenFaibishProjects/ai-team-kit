list_alt Step-by-Step Guide looks_one Access Project Creation

From the Dashboard:

  1. Navigate to your dashboard
  2. Look for the "Start New Project" button
  3. Click to begin the project creation wizard
lightbulb

Tip: You can also start a project from the Projects page by clicking the "+" button.

 looks_two Enter Project Information

Required Information:

  • Project Name: Choose a clear, descriptive name
  • Description: Explain the project's purpose and goals
  • Start Date: When the project begins
  • Target End Date: Expected completion date
  • Budget: Allocated resources (optional)

Example:

Name: "E-Commerce Platform Redesign"

Description: "Modernize our online store with improved UX, mobile responsiveness, and faster checkout process"

 looks_3 Configure Team Settings

Team Configuration:

  1. Select team members from your organization
  2. Assign roles (Product Owner, Scrum Master, Developers, etc.)
  3. Set team capacity and availability
  4. Define communication channels
warning

Important: Ensure all team members have been added to your organization before starting the project.

 looks_4 Define Features & Requirements

Feature Planning:

  • List all major features and user stories
  • Prioritize using MoSCoW method (Must have, Should have, Could have, Won't have)
  • Estimate effort for each feature
  • Group related features into epics
lightbulb

Pro Tip: Use the AI Command Center to generate user stories and acceptance criteria automatically!

 looks_5 Set Up Sprint Planning

Sprint Configuration:

  1. Define sprint duration (typically 1-4 weeks)
  2. Set sprint goals and objectives
  3. Allocate features to sprints
  4. Establish sprint ceremonies (daily standups, reviews, retrospectives)

Recommended Sprint Structure:

  • Sprint Duration: 2 weeks
  • Daily Standup: 15 minutes, same time daily
  • Sprint Review: Last day of sprint, 1 hour
  • Sprint Retrospective: After review, 45 minutes
 looks_6 Review & Launch

Final Checklist:

Project information complete Team members assigned Features defined and prioritized Sprint schedule configured Communication channels set up Stakeholders notified

Ready to launch?

Click "Create Project" to finalize and start your project!

 help_outline Frequently Asked Questions How long should my first sprint be?

For new teams, we recommend starting with 2-week sprints. This provides enough time to complete meaningful work while maintaining frequent feedback cycles. You can adjust the duration based on your team's experience and project needs.

 Can I modify project details after creation?

Yes! You can edit project information, add/remove team members, and adjust features at any time. Navigate to the project settings to make changes.

 What if I don't have all features defined yet?

That's perfectly fine! Start with the features you know and add more as they become clear. Agile methodology embraces evolving requirements. You can use the AI Command Center to help brainstorm and define features.

 How do I integrate with GitHub?

After creating your project, go to Project Settings > Integrations > GitHub. Follow the authentication flow to connect your repository. This enables automatic issue creation and progress tracking.

 Can I use templates for common project types?

Yes! The AI Command Center provides templates for various project types (web apps, mobile apps, APIs, etc.). These templates include pre-defined features, user stories, and best practices specific to each project type.
